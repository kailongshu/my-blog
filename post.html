深入理解 JavaScript 闭包与作用域链

👨‍💻 小明

闭包（Closure）是 JavaScript 中最重要也最容易被误解的概念之一。理解闭包不仅是面试中的高频考点，更是写出高质量 JavaScript 代码的基石。这篇文章将从基础概念开始，带你深入闭包的世界。

一、什么是作用域？

在理解闭包之前，我们必须先搞清楚作用域（Scope）的概念。作用域决定了变量在代码中的可访问范围。JavaScript 中有三种作用域：

  • 全局作用域：在任何函数外部声明的变量，代码中的任何位置都可以访问。
  • 函数作用域：在函数内部声明的变量，只能在函数内部访问。
  • 块级作用域（ES6+）：用 let 和 const 声明的变量，只在最近的 { } 代码块内有效。
// 作用域示例
const globalVar = '我是全局变量';

function outer() {
    const outerVar = '我在 outer 函数内';

    function inner() {
        const innerVar = '我在 inner 函数内';
        console.log(globalVar);  // ✅ 可访问
        console.log(outerVar);   // ✅ 可访问
        console.log(innerVar);   // ✅ 可访问
    }
    inner();
    console.log(innerVar);     // ❌ ReferenceError
}
outer();

二、词法作用域与作用域链

JavaScript 使用词法作用域（Lexical Scoping），这意味着函数的作用域在函数定义时就已经确定，而不是在函数调用时确定的。当 JavaScript 引擎查找一个变量时，它会沿着作用域链逐级向上搜索——从当前作用域开始，一直到全局作用域为止。

可以把作用域链想象成一个嵌套的抽屉柜：最内层的抽屉是当前函数的作用域，如果找不到想要的变量，就打开外面一层的抽屉，直到最外层的大柜子（全局作用域）。

词法作用域是闭包能够工作的根本原因。因为内部函数「记住」了它被定义时的环境，所以即使外部函数已经执行完毕，内部函数仍然可以访问外部函数的变量。

三、闭包的核心定义

闭包（Closure）是指一个函数能够记住并访问它的词法作用域，即使该函数在它的词法作用域之外被执行。

通俗来说：当一个函数「记住」了它出生时的环境，就形成了闭包。

// 经典闭包示例
function createCounter() {
    let count = 0;  // 这个变量会被闭包"记住"

    return function() {
        count++;  // 访问了外部函数的变量
        return count;
    };
}

const counter = createCounter();
console.log(counter());  // 1
console.log(counter());  // 2
console.log(counter());  // 3

在这个例子中，createCounter 执行后返回的内部函数，仍然可以访问 count 变量。即使 createCounter 已经执行完毕、它的执行上下文已经从调用栈中弹出，count 变量依然「存活」在内存中，因为返回的内部函数持有对它的引用。

四、闭包的实际应用场景

4.1 数据私有化（模块模式）

闭包可以用来创建私有变量，对外只暴露有限的 API：

const bankAccount = (function() {
    let balance = 0;  // 私有变量

    return {
        deposit(amount) {
            balance += amount;
            return balance;
        },
        withdraw(amount) {
            if (amount > balance) {
                return '余额不足';
            }
            balance -= amount;
            return balance;
        },
        getBalance() {
            return balance;
        }
    };
})();

4.2 函数工厂

创建一组功能类似但行为不同的函数：

function makeMultiplier(factor) {
    return function(number) {
        return number * factor;
    };
}

const double = makeMultiplier(2);
const triple = makeMultiplier(3);

console.log(double(5));  // 10
console.log(triple(5));  // 15

4.3 事件处理与异步编程

在循环中使用闭包来"捕获"每次迭代的值：

// ❌ 错误写法 — 所有点击事件打印同一个值
for (var i = 1; i <= 5; i++) {
    button.addEventListener('click', () => {
        console.log(i);  // 总是 6
    });
}

// ✅ 正确写法 — let 创建块级作用域
for (let i = 1; i <= 5; i++) {
    button.addEventListener('click', () => {
        console.log(i);  // 1, 2, 3, 4, 5
    });
}

五、注意事项与性能考量

闭包虽然强大，但也有一些需要注意的地方：

  • 内存占用：闭包会保持对外部变量的引用，导致它们无法被垃圾回收。不需要时应及时解除引用。
  • 避免过度使用：不是所有场景都需要闭包。如果简单的参数传递就能解决问题，就不必创建闭包。
  • 循环中的闭包陷阱：使用 let 替代 var 通常是解决循环闭包问题的最简单方案。

六、总结

闭包不是 JavaScript 的「黑魔法」—— 它只是词法作用域的自然结果。理解作用域和作用域链是理解闭包的前提。一旦掌握了闭包，你就解锁了 JavaScript 中更高级的编程模式，包括模块化、函数式编程、高阶函数等。
